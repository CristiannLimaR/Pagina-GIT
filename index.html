Guía de Comandos Git

¿Qué es Git?

Git es un sistema de control de versiones open Source, gratuito, diseñado para manejar todo, desde proyectos pequeños a muy grandes, permitiendo el trabajo colaborativo.

Comandos Git

git init

El comando git init se utiliza para crear un nuevo repositorio de Git. Esto inicializa un nuevo subdirectorio llamado .git que contiene todos los archivos necesarios del repositorio, esencialmente creando una nueva instancia de un repositorio de Git en el directorio actual.

Ejemplo: git init

git remote add

El comando git remote add permite añadir un nuevo repositorio remoto. Esto es útil para trabajar en colaboración, ya que puedes añadir la URL de un repositorio remoto y darle un nombre para poder hacer referencia a él fácilmente en comandos posteriores.

Ejemplo: git remote add origin https://github.com/CristiannLimaR/SuperKinal.git

git add

El comando git add se utiliza para añadir archivos al área de preparación. Esto indica a Git que deseas incluir las actualizaciones de esos archivos en el próximo commit. Puedes añadir archivos individuales, directorios completos o utilizar patrones para seleccionar múltiples archivos a la vez.

Ejemplo: git add archivo.txt

git status

El comando git status muestra el estado actual del repositorio, incluyendo los archivos que están en el área de preparación, los que no están rastreados y los que han sido modificados. Esto es útil para ver qué cambios se han realizado y qué archivos están listos para ser confirmados.

Ejemplo: git status

git branch

El comando git branch permite gestionar las ramas del repositorio. Puedes crear nuevas ramas, listar las ramas existentes y eliminar ramas que ya no necesitas. Trabajar con ramas facilita el desarrollo de nuevas funcionalidades sin afectar la rama principal del proyecto.

Ejemplo: git branch

git checkout

El comando git checkout se utiliza para cambiar entre ramas o restaurar archivos en el directorio de trabajo. Al cambiar de rama, Git actualiza los archivos del directorio de trabajo para reflejar la versión más reciente de la rama seleccionada.

Ejemplo: git checkout rama

git pull

El comando git pull descarga cambios desde un repositorio remoto y los fusiona con la rama actual del repositorio local. Esto asegura que tu repositorio local esté actualizado con los últimos cambios realizados por otros colaboradores.

Ejemplo: git pull origin master

git push

El comando git push se utiliza para subir los cambios locales a un repositorio remoto. Esto hace que tus commits locales sean accesibles para otros colaboradores en el repositorio remoto. Puedes especificar la rama que deseas empujar.

Ejemplo: git push origin master

git merge

El comando git merge se utiliza para fusionar una rama especificada con la rama actual. Esto integra los cambios de diferentes ramas en una sola. Es útil para combinar el trabajo de múltiples desarrolladores.

Ejemplo: git merge rama

git commit

El comando git commit guarda los cambios en el historial del repositorio. Cada commit incluye un mensaje que describe los cambios realizados. Esto crea un punto en el historial del proyecto al que se puede volver si es necesario.

Ejemplo: git commit -m "Mensaje de commit"
